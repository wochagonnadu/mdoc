# Документация по использованию CLI-утилиты `mdoc`

## Описание

Утилита `mdoc` предназначена для генерации и агрегации README-файлов на разных уровнях модулей проекта. Она помогает автоматически создавать документацию для "листовых" модулей и собирать её в агрегированные файлы в родительских директориях, что упрощает поддержку и обновление документации в больших проектах.

## Установка

Рекомендуется устанавливать `mdoc` с помощью `pipx`, чтобы избежать конфликтов зависимостей и сделать утилиту доступной глобально.

```bash
# Установка из локальной директории
pipx install .

# Или установка напрямую из Git-репозитория
# pipx install git+https://github.com/your_username/mdoc.git
```

После установки команда `mdoc` будет доступна в вашей системе.

## Использование

```bash
mdoc <команда> [опции]
```

## Команды

### `dump-leaves`

- **Назначение:** Генерирует документацию для листовых (нижних) модулей проекта. Создаёт файлы `module_dump.md` и `README.generated.md` в соответствующих папках.
- **Пример запуска:**

```bash
# Запуск из корня проекта
mdoc dump-leaves .
```

- **Что создаёт:**
  - В папках с листовыми модулями создаются файлы:
    - `module_dump.md` — подробный дамп содержимого модуля, включающий полный листинг кода с комментариями.
    - `README.generated.md` — автоматически сгенерированный README для данного модуля, содержащий описание назначения модуля, его функций, классов и структуры.

### `aggregate`

- **Назначение:** Собирает документацию из дочерних модулей и подготавливает материалы для сводного README верхнего уровня.
- **Пример запуска:**

```bash
# Запуск в папке, где нужно агрегировать README
mdoc aggregate .
```

- **Что делает и что создаёт:**
  - Читает в каждой дочерней папке файлы:
    - `README.generated.md` — автосгенерированный README (если есть)
    - `README.md` — ручной README (если есть)
  - Объединяет найденное содержимое и записывает в корневой для запуска файл:
    - `module_aggregated.md` — «сырые материалы» для сводной документации. Если ничего не найдено, файл содержит понятный заглушечный текст вместо пустого содержимого.
  - Если отсутствует, создаёт файл-шаблон:
    - `README.aggregated.md` — сводный README, который должен быть сформирован человеком или LLM на основе `module_aggregated.md`.

## Типичный рабочий процесс

1. Запустите команду `dump-leaves` для генерации документации в листовых модулях.
2. Запустите команду `aggregate` в родительских папках — это создаст/обновит `module_aggregated.md` и при необходимости сгенерирует шаблон `README.aggregated.md`.
3. На основе `module_aggregated.md` подготовьте (вручную или с помощью LLM) содержимое `README.aggregated.md`.
4. При необходимости повторяйте шаги для всех уровней модулей, поднимаясь от листьев к корню проекта.

Таким образом, документация автоматически собирается и обновляется на всех уровнях структуры проекта.

**Примечание:** на верхнем уровне после нескольких проходов будет собрана иерархия документации, в которой каждый уровень содержит агрегированное описание нижележащих модулей.
