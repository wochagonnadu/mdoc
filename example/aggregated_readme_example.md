# README.aggregated.md

Этот файл содержит сводную информацию о модулях, расположенных в директории `services`.
Он предназначен для быстрого ознакомления с основными функциями, зависимостями и публичными интерфейсами подсистемы.

## Обзор модулей

### Модуль `auth`

**Назначение:** Управление аутентификацией пользователей, регистрацией и сессиями.
**Публичные интерфейсы:** `login`, `logout`, `register_user`, `get_current_user`.
**Ключевые зависимости:** `core.database`, `fastapi`, `passlib`, `pyjwt`.
**Конфигурация:** `JWT_SECRET_KEY`, `DATABASE_URL`, `config.yml`.
**Тесты:** Присутствуют, `test_auth.py`.

[Подробнее о модуле `auth`](auth/README.generated.md)

### Модуль `payment` (гипотетический)

**Назначение:** Обработка платежей и управление транзакциями.
**Публичные интерфейсы:** `create_payment`, `process_webhook`, `get_payment_status`.
**Ключевые зависимости:** `core.database`, `yookassa` (гипотетически).
**Конфигурация:** `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY`, `payment_config.json`.
**Тесты:** Присутствуют, `test_payment.py`.

[Подробнее о модуле `payment`](payment/README.generated.md)

## Общие зависимости

Подсистема `services` в целом зависит от общей базы данных (`core.database`) и использует общие механизмы логирования и обработки ошибок.

## Как использовать

Для взаимодействия с модулями `auth` и `payment` используйте их публичные API, как описано в соответствующих `README.generated.md` файлах.

## Развертывание и запуск

Модули развертываются как часть общего бэкенд-сервиса. Конкретные инструкции по развертыванию и запуску находятся в корневом `README.md` проекта.
