


# Многоуровневый флоу генерации README

## Шаг 1 — Генерация для нижнего уровня (leaf-модули)
**Задача:**
Идём в каждую «конечную» папку, где лежит только код (нет поддиректорий с модулями).
Для каждой такой папки создаём:
- `module_dump.md` — с кодом и структурой.
- `README.generated.md` — каркас.
- Заполняем `README.md` (на базе `module_dump.md`).

**Результат:**
У каждого leaf-модуля есть полноценный `README.md`.

---

## Шаг 2 — Агрегация второго уровня
**Задача:**
Для каждой родительской папки, у которой в поддиректориях уже есть готовые `README.md`, создаём `module_aggregated.md`.

В `module_aggregated.md` включаем:
1. Дерево подмодулей.
2. Из каждого подмодуля вытягиваем:
   - `purpose` из YAML-хинтов его README.
   - Ссылку на README.
3. (Опционально) куски Mermaid-схем из подмодулей.

**Важно:**
На этом уровне код уже не тащим, только данные из готовых README нижнего уровня.

---

## Шаг 3 — Заполнение README для второго уровня
**Задача:**
На базе `module_aggregated.md` делаем `README.md` для второго уровня.
Он описывает:
- Общую роль этой папки.
- Таблицу подмодулей (название / назначение / ссылка).
- Схему взаимодействий между подмодулями.

---

## Шаг 4 — Повтор для каждого следующего уровня
Для уровня №3 собираем `module_aggregated.md` из README второго уровня.
Делаем `README.md` уровня №3.
И так до самого верха (например, `backend/api` или даже весь `backend`).

---

## Итоговая схема
```
LEAF: code → module_dump.md → README.md
↑
LEVEL 2: README нижних → module_aggregated.md → README.md
↑
LEVEL 3: README уровня 2 → module_aggregated.md → README.md
↑
...
```

---

## Что нужно для автоматизации
1. `dump_leaf_modules.sh` — ищет leaf-папки, делает `module_dump.md` и `README.generated.md`.
2. `aggregate_level.sh` — берёт все `README.md` из подпапок и создаёт `module_aggregated.md` для текущей папки.
3. Промпт для LLM — по `module_aggregated.md` генерирует `README.md` уровня.
